Prefer `decl_search`, `reference_search`, or `symbol_context` for symbol-shaped queries in Rust, Python, Java, C#, Go, C/C++, JavaScript/TypeScript, Ruby, PHP, Kotlin, Swift, Scala, and Dart files when the query is a bare name and the languages have nominal type systems (Rust, Go, C#, C++, Java, Kotlin, Swift, Scala, Dart) — they follow imports, qualified paths, and re-exports that regex misses. For Python, JS/TS, Ruby, or PHP aliased-import resolution, or any multi-symbol tabulation (count usages across N symbols, aggregate fields per class), `glob` with a focused query or script is usually one shot. Use `glob` for literal text (strings, comments, config keys) or files outside Rust, Python, Java, C#, Go, C/C++, JavaScript/TypeScript, Ruby, PHP, Kotlin, Swift, Scala, and Dart. List workspace file paths matching a glob without reading file contents. Respects .gitignore by default; set include_ignored=true only when ignored paths are intentionally needed.
